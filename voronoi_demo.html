What's Been Added

A visual overlay system that uses Voronoi diagrams to improve trail selection accuracy when multiple trails overlap on the map.

How to Use

  • Q - Toggle the Voronoi overlay on/off
  • 1 - Use all trail points (highest precision, slowest)
  • 2 - Use every 2nd point
  • 3 - Use every 4th point
  • 4 - Use every 8th point
  • 5 - Use every 16th point (lowest precision, fastest)
  • ...and so on up to 9 (every 256th point)

Problem Solved

‚ùå Before

Issue: Multiple routes selected with one tap/click

  • Overlapping trails cause ambiguity
  • 20px tolerance zone selects all nearby trails
  • Hard to select just one specific trail

‚úÖ After

Solution: Voronoi-based precise selection

  • Each point on the map belongs to exactly one trail
  • Always selects the closest trail
  • Visual feedback shows selection regions

Visual Elements

When enabled, the overlay shows:

  • Colored cells - Semi-transparent regions showing which trail each area belongs to
  • Black dots - Sampled points from trail coordinates used to generate the diagram
  • Cell boundaries - Lines showing the Voronoi cell edges
  • 15 distinct colors - Helps distinguish between different trails

Technical Details

  • Library: d3-delaunay for efficient Voronoi computation
  • Rendering: HTML5 Canvas for smooth performance
  • Selection: O(log n) nearest point lookup using Delaunay triangulation
  • Optimization: 2^n point sampling (1, 2, 4, 8, 16, 32, 64, 128, 256)
  • Auto-update: Regenerates when trails change or map moves
